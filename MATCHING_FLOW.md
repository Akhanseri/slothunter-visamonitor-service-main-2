# 🔄 Полный flow Matching системы

## 📊 Визуализация процесса

```
┌─────────────────────────────────────────────────────────────┐
│ 1️⃣  СОЗДАНИЕ КЛИЕНТА                                        │
├─────────────────────────────────────────────────────────────┤
│ POST /api/clients/manual                                    │
│ {                                                           │
│   email, password, isResident,                              │
│   groups: [{status, schedulePath}]                          │
│ }                                                           │
│                                                             │
│ Результат в БД:                                             │
│ ┌─────────────────────────────┐                             │
│ │ Client                      │                             │
│ │  queueIndex: 1       ← авто│                             │
│ │  lastProcessedAt: null      │                             │
│ └─────────────────────────────┘                             │
│ ┌─────────────────────────────┐                             │
│ │ ClientVisaGroup             │                             │
│ │  city: null          ← НЕТ! │                             │
│ │  slotStartDate: null ← НЕТ! │                             │
│ │  slotEndDate: null   ← НЕТ! │                             │
│ │  delayDays: null     ← НЕТ! │                             │
│ │  matchStatus: null   ← НЕТ! │                             │
│ └─────────────────────────────┘                             │
│                                                             │
│ ⚠️ Parser НЕ обработает (city=null)                         │
└─────────────────────────────────────────────────────────────┘

                          ↓

┌─────────────────────────────────────────────────────────────┐
│ 2️⃣  НАСТРОЙКА MATCHING                                      │
├─────────────────────────────────────────────────────────────┤
│ POST /api/clients/1/visa-groups/setup-matching              │
│ {                                                           │
│   visaGroupId: 1,                                           │
│   city: "astana",                                           │
│   slotStartDate: "15.12",                                   │
│   slotEndDate: "31.12",                                     │
│   delayDays: 5,                                             │
│   isAutoBookEnabled: false                                  │
│ }                                                           │
│                                                             │
│ Обновление в БД:                                            │
│ ┌─────────────────────────────┐                             │
│ │ ClientVisaGroup             │                             │
│ │  city: "astana"       ← ✅  │                             │
│ │  slotStartDate: "15.12" ✅  │                             │
│ │  slotEndDate: "31.12"   ✅  │                             │
│ │  delayDays: 5           ✅  │                             │
│ │  matchStatus: "NEW"     ✅  │                             │
│ │  isAutoBookEnabled: false   │                             │
│ └─────────────────────────────┘                             │
│                                                             │
│ ✅ Готово к парсингу!                                       │
└─────────────────────────────────────────────────────────────┘

                          ↓

┌─────────────────────────────────────────────────────────────┐
│ 3️⃣  PARSER ОБРАБОТКА (каждые ~15 минут)                    │
├─────────────────────────────────────────────────────────────┤
│ ClientQueueManager.getNextClient("resident")                │
│   ↓                                                         │
│ ✅ Выбран клиент (queueIndex=1)                             │
│   ↓                                                         │
│ Авторизация на ais.usvisa-info.com                          │
│   ↓                                                         │
│ Парсинг слотов (300 итераций):                              │
│ [                                                           │
│   {city: "astana", date: "20.12.2025", time: "10:00"},      │
│   {city: "astana", date: "22.12.2025", time: "14:00"},      │
│   {city: "almaty", date: "18.12.2025", time: "11:00"}       │
│ ]                                                           │
│   ↓                                                         │
│ _processSlotsForClient(slots, client)                       │
│   ↓                                                         │
│ ┌───────────────────────────────────────────────┐           │
│ │ Для каждой active visa group:                │           │
│ │                                               │           │
│ │ 1. Фильтр по городу:                          │           │
│ │    city="astana" → 2 слота                    │           │
│ │                                               │           │
│ │ 2. Построить окно:                            │           │
│ │    today: 10.12.2025                          │           │
│ │    delayDays: 5                               │           │
│ │    → minDate: 15.12.2025                      │           │
│ │    → window: 15.12 - 31.12                    │           │
│ │                                               │           │
│ │ 3. Найти первый matching:                     │           │
│ │    20.12.2025 >= 15.12 ✅                     │           │
│ │    20.12.2025 <= 31.12 ✅                     │           │
│ │    → MATCH! 🎉                                │           │
│ │                                               │           │
│ │ 4. Обновить БД:                               │           │
│ │    matchStatus: "MATCH_PENDING"               │           │
│ │    candidateSlot: {                           │           │
│ │      city: "astana",                          │           │
│ │      date: "20.12.2025",                      │           │
│ │      time: "10:00"                            │           │
│ │    }                                          │           │
│ │    candidateSlotExpiresAt: NOW + 1 hour       │           │
│ │                                               │           │
│ │ 5. Отправить Telegram уведомление 📱          │           │
│ └───────────────────────────────────────────────┘           │
└─────────────────────────────────────────────────────────────┘

                          ↓

┌─────────────────────────────────────────────────────────────┐
│ 4️⃣  ПОЛЬЗОВАТЕЛЬ ПОЛУЧАЕТ УВЕДОМЛЕНИЕ                      │
├─────────────────────────────────────────────────────────────┤
│ Telegram сообщение:                                         │
│ ┌─────────────────────────────────────────┐                 │
│ │ 🎯 Найден подходящий слот!              │                 │
│ │                                         │                 │
│ │ Клиент: user@example.com                │                 │
│ │ Город: astana                           │                 │
│ │ Дата: 20.12.2025                        │                 │
│ │ Время: 10:00                            │                 │
│ │                                         │                 │
│ │ Записаться до: [время истечения]        │                 │
│ │                                         │                 │
│ │ [Кнопка: Записаться] ← API endpoint     │                 │
│ └─────────────────────────────────────────┘                 │
└─────────────────────────────────────────────────────────────┘

                          ↓

┌─────────────────────────────────────────────────────────────┐
│ 5️⃣  ЗАПИСЬ НА СЛОТ (опционально)                           │
├─────────────────────────────────────────────────────────────┤
│ Вариант А: Автоматическая (isAutoBookEnabled=true)         │
│   → Parser сразу бронирует слот                             │
│   → matchStatus: "BOOKED"                                   │
│                                                             │
│ Вариант Б: Ручная (isAutoBookEnabled=false)                │
│   → POST /api/clients/1/visa-groups/book                    │
│   → matchStatus: "BOOKING_IN_PROGRESS" → "BOOKED"           │
└─────────────────────────────────────────────────────────────┘
```

---

## ✅ Проверка логики

### Все проверки пройдены:

✅ **setup-matching endpoint**
- Сохраняет параметры ✅
- Сбрасывает статус на NEW ✅
- Очищает старые candidateSlot ✅
- Защита от изменений во время booking ✅

✅ **Parser matching логика**
- Фильтрует только активные группы с параметрами ✅
- Правильно строит временное окно ✅
- Учитывает delayDays ✅
- Фильтрует по городу (case-insensitive) ✅
- Использует бинарный поиск (эффективно) ✅
- Находит самый ранний слот в окне ✅

✅ **Обновление статусов**
- NEW → MATCH_PENDING (когда найден) ✅
- MATCH_PENDING → BOOKED (при записи) ✅
- Обновляет candidateSlot правильно ✅
- Отправляет Telegram уведомления ✅

## 🎯 Итого

**ВСЯ ЛОГИКА РАБОТАЕТ ПРАВИЛЬНО!** ✅

Единственное требование: **ОБЯЗАТЕЛЬНО вызвать setup-matching** после создания клиента, иначе parser не будет обрабатывать группу (city=null).

Но текущая логика **полностью корректна и работоспособна!** ✅
