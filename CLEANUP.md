# 🧹 Автоматическая очистка системы

## Описание

Скрипты для автоматической очистки неиспользуемых данных на сервере, предотвращающие заполнение диска и ошибки работы PostgreSQL.

## Скрипты

### `cleanup-system.sh`

Комплексная очистка системы:
1. Системные логи journald (старше 7 дней)
2. Логи приложения (старше 30 дней)
3. Временные файлы (старше 7 дней)
4. Кэш Puppeteer (старше 30 дней)

**Использование:**
```bash
./cleanup-system.sh
```

## Автоматический запуск (Cron)

Скрипты настроены на автоматический запуск через cron:

```bash
# Просмотр текущих задач
crontab -l

# Редактирование задач
crontab -e
```

### Расписание

- **Ежедневная очистка системы**: каждый день в 3:00 ночи
  ```
  0 3 * * * /root/visamonitor/cleanup-system.sh >> /root/visamonitor/logs/cleanup.log 2>&1
  ```

## Логи

Логи очистки сохраняются в:
```
/root/visamonitor/logs/cleanup.log
```

## Мониторинг использования диска

Проверка использования диска:
```bash
df -h /
```


Проверка размера логов:
```bash
journalctl --disk-usage
du -sh /root/visamonitor/logs
```

## Ручная очистка при необходимости

Если диск заполнен более чем на 80%, можно запустить очистку вручную:

```bash
# Комплексная очистка
/root/visamonitor/cleanup-system.sh
```

## Предупреждения

Скрипты автоматически предупреждают, если:
- Диск заполнен более чем на 80%
- Не удалось освободить место

В таких случаях рекомендуется проверить вручную:
```bash
du -h --max-depth=1 /var | sort -hr | head -10
du -h --max-depth=1 /root | sort -hr | head -10
```

## Безопасность

Скрипты:
- ✅ Не удаляют активные процессы
- ✅ Не удаляют данные, используемые менее указанного времени
- ✅ Сохраняют логи всех операций
- ✅ Безопасны для работы приложения
